1. Verify GitHub Pages Configuration

  1. Navigate to Your Repository:

  3. Locate GitHub Pages Section:

  4. Verify Source:

      • Branch: main (or master) branch.
      • Folder: /root (root of the branch) or /docs folder.

b. Save and Wait:

  • After making any changes, click Save and wait a few minutes for GitHub Pages to rebuild your site.

2. Ensure Presence of index.html

GitHub Pages looks for an index.html file in the root (or specified folder) to serve as the homepage. If it doesn't find one, it may default to displaying the README.md.

a. Check Repository Structure:

  • Ensure that an index.html file exists in the branch and folder you've set as the source for GitHub Pages.

b. If Using a Frontend Framework:

  • React (Create React App):
    • Run npm run build to generate the build folder.
    • Ensure the contents of the build folder (including index.html) are in the correct location (e.g., /docs if that's your source).
  • Vue.js, Angular, etc.:
    • Similar to React, ensure the build output is correctly placed.

3. Check Build and Deployment Process

If you're using a static site generator or a build tool, ensure that the build process completes successfully and outputs the necessary files.

a. Local Testing:

  • Run your application locally to ensure it builds and runs without errors.

b. Automated Deployments:

  • If using GitHub Actions or another CI/CD tool, check the workflow logs to ensure the build and deployment steps are completing successfully.

4. Review Repository Contents

Ensure that there are no conflicting files that might cause GitHub Pages to default to the README.

a. Absence of index.html:

  • If index.html is missing, GitHub Pages will display the README.md.

b. File Placement:

  • index.html should be in the root of the selected branch/folder, not nested inside subdirectories unless configured accordingly.

5. Check for Custom Domain Issues

If you're using a custom domain, misconfigurations can lead to display issues.

a. DNS Settings:

  • Ensure that your DNS records (A, CNAME) are correctly pointed to GitHub Pages.

b. GitHub Repository Settings:

  • In the GitHub Pages section, ensure that the custom domain is correctly set.

c. HTTPS Configuration:

  • Ensure that "Enforce HTTPS" is enabled if applicable.

6. Clear Browser Cache

Sometimes, browsers cache the previous state of your site.

a. Clear Cache:

  • Clear your browser's cache or try accessing your site in an incognito/private window.

7. Wait for DNS and Build Propagation

After making changes, especially DNS-related, it might take some time for the changes to propagate.

a. Patience:

  • Wait up to 10 minutes (sometimes longer) for GitHub Pages to build and serve your site.

8. Inspect GitHub Pages Build Logs

GitHub provides build logs that can help identify issues during the site generation process.

a. Access Build Logs:

  • In the GitHub Pages section of your repository's settings, look for any build status or error messages.

9. Example Steps to Set Up GitHub Pages Correctly

  2. Change npm’s Default Directory: Configure npm to use a directory within your home folder for global installations.

bash

bash
npm config set prefix "${HOME}/.npm-global"

    bash
